OpenAVL

A numerically faithful Python port of AVL (Athena Vortex Lattice) — the classical vortex-lattice aerodynamic analysis code by Mark Drela and Harold Youngren (MIT).

Python 3.10+ NumPy / SciPy JAX-ready GPL v2+

Overview

OpenAVL is a pure Python reimplementation of the Fortran AVL solver. It provides the same vortex-lattice aerodynamics computations as the original code while exposing a clean, object-oriented API suitable for scripting, optimization loops, and future automatic differentiation via JAX.

What OpenAVL does

  • Parses .avl geometry files describing surfaces, sections, control surfaces, and airfoils.
  • Builds aircraft programmatically via the Geometry API (Aircraft, Wing, Section) without writing .avl files.
  • Assembles a vortex-lattice panel mesh over every lifting surface and slender body.
  • Solves the circulation distribution using an Aerodynamic Influence Coefficient (AIC) matrix.
  • Computes total forces (CL, CD, CY), moments (CMx, CMy, CMz), and far-field Trefftz-plane quantities.
  • Supports multi-variable trim/constraint solving via a Newton iteration loop, with setup_trim() presets for level flight and pull-ups.
  • Loads mass and inertia from .mass files for trim and flight-dynamics analysis.
  • Extracts stability derivatives with respect to alpha, beta, rotation rates, and control deflections.
  • Performs linearized flight-dynamics eigenanalysis (short period, phugoid, Dutch roll, spiral, etc.).
  • Applies piecewise-quadratic CD(CL) viscous drag polars per surface or section.
  • Caps sectional lift coefficients per wing via clmax (Geometry API only) to approximate stall onset and its effect on integrated forces and moments.
  • Visualizes aircraft geometry, spanwise lift distributions, and pressure coefficient (Cp) via AVLSolver.plot_aircraft(), plot_lift_distribution(), and plot_cp().

What OpenAVL does not do (yet)

  • Interactive GUI (openavl.gui is a placeholder namespace only).
  • JAX automatic differentiation (planned for a future phase).

Optional extras

Extras use bracket notation and can be combined (e.g. pip install -e ".[dev,jax,web]"). 

pip install -e ".[dev]"         # pytest
pip install -e ".[web]"         # FastAPI + uvicorn + python-multipart
pip install -e ".[jax]"         # JAX + jaxlib (CPU)
pip install -e ".[jax-cuda12]"  # JAX with CUDA 12 GPU support

See Web GUI and JAX below for what each extra provides and how to verify the install.

Web GUI (Optional)

The browser-based GUI lives under openavl.web. It wraps the core NumPy solver with a FastAPI backend and WebSocket session handling. It is not installed with the base package.

Install

pip install -e ".[web]"

Pulls in fastapi, uvicorn[standard], and python-multipart (for airfoil file uploads).

Launch

python -m openavl.web

Starts the server on http://127.0.0.1:8000 and opens it in your default browser. Matplotlib-based plotting (openavl.plotting) works without this extra.

Run web tests

pytest tests/ui

JAX (Optional)

OpenAVL ships a JAX backend under openavl.jax for automatic differentiation, gradient-based optimization, and OpenMDAO integration (see examples/rectangular_wing_opt.py). JAX is not installed with the base package.

CPU install

pip install -e ".[jax]"

This pulls in jax and jaxlib for CPU execution.

Windows with NVIDIA GPU (CUDA 12)

On Windows, JAX requires native DLLs from jaxlib. If imports fail with DLL or ImportError messages, install the Microsoft Visual C++ Redistributable (x64) from Microsoft, then install JAX with CUDA 12 support: 

# Upgrade/install JAX with CUDA 12 wheels (includes jaxlib)
pip install -U "jax[cuda12]"

# Or via the OpenAVL extra:
pip install -e ".[jax-cuda12]"
Windows note Use a standard Python venv rather than Anaconda if jaxlib still fails to load after installing the VC++ redistributable. WSL is another fallback if native Windows wheels remain problematic.

Quick Start

The simplest workflow: load a geometry file, set flight conditions, run the solver, and read the results. 

from openavl import AVLSolver

# 1. Load geometry
solver = AVLSolver("tests/data/avl/geometries/plane.avl")

# 2. Set flight conditions
solver.set_variable("alpha", 2.0)         # angle of attack in degrees
solver.set_variable("beta", 0.0)          # sideslip angle in degrees
solver.set_parameter("mach", 0.2)
solver.set_parameter("velocity", 50.0)    # m/s
solver.set_parameter("density", 1.225)   # kg/m^3

# 3. Run the Newton solver
solver.execute_run(max_iter=20)

# 4. Retrieve results
results = solver.get_results()
print(f"CL = {results['CL']:.4f}")
print(f"CD = {results['CD']:.4f}")
print(f"Converged: {results['converged']}")

Constructor keyword shortcuts

Common run-case values can be passed directly to the constructor instead of calling set_parameter / set_variable afterward: 

solver = AVLSolver(
    "tests/data/avl/geometries/supra.avl",
    mass_file="tests/data/avl/mass/supra.mass",
    alpha=2.0,       # degrees
    vel=15.0,         # airspeed (m/s)
    rho=1.225,
    gravity=9.81,
    cd0=0.015,
    xcg=3.75,
)

Trim example: fly at a target CL

Use set_constraint() to let the solver find the alpha that produces a desired lift coefficient, instead of prescribing alpha directly. 

from openavl import AVLSolver

solver = AVLSolver("tests/data/avl/geometries/plane.avl")
solver.set_parameter("mach", 0.15)

# Drive alpha to satisfy CL = 0.5
solver.set_constraint("alpha", "cl", 0.5)
solver.execute_run(max_iter=30)

r = solver.get_results()
print(f"alpha = {r['alpha_deg']:.3f} deg  CL = {r['CL']:.4f}")

Level-flight trim with setup_trim()

For coordinated maneuvers, call setup_trim(mode=1) before execute_run(). Mode 1 presets longitudinal constraints for level flight or a banked turn (load factor from bank angle). Mode 2 configures a steady pull-up with prescribed turn radius. Reapply lateral constraints (beta, aileron, rudder) after setup_trim if needed. 

solver.set_parameter("cl", 0.7)
solver.setup_trim(mode=1)
solver.set_constraint("elevator", "cm", 0.0)
solver.execute_run()

Example with control surface deflection

solver = AVLSolver("tests/data/avl/geometries/plane.avl")
solver.set_variable("alpha", 3.0)

# Deflect a named control surface (name must match the CONTROL card in the .avl file)
solver.set_variable("elevator", 5.0)   # +5 degrees trailing-edge down

solver.execute_run()
print(solver.get_results())

AVLSolver

AVLSolver is the primary user-facing class. Import it from the top-level package: 

from openavl import AVLSolver

Legacy imports (openavl.solver, openavl.parser) remain as thin re-export shims.

Constructor

AVLSolver(geo: str | Path | Aircraft, mass_file: str | Path | None = None, debug: bool = False, **state_options)
ParameterTypeDescription
geostr | Path | AircraftPath to a .avl file, or a programmatic Aircraft instance from the Geometry API.
mass_filestr | Path | NoneOptional path to a .mass file. Required for weight-based trim and eigenanalysis.
debugboolEnable verbose debug output from the solver. Default False.
**state_optionskeyword argsInitial run-case values: alpha, beta (degrees), vel, rho, gravity, cd0, xcg/ycg/zcg, mach, etc. When using an Aircraft, pass base_dir to resolve relative airfoil/body file paths.

The constructor parses the geometry file, allocates all solver arrays in an AVLState, and builds the vortex-lattice mesh. After construction the solver is ready to accept variables, parameters, and constraints.

Methods

None  set_variable(name: str, value: float) → None

Set a run-case variable — a quantity the solver will use as a fixed input or as the unknowns it adjusts during constraint solving. Angles are in degrees; rotation rates are in the non-dimensional forms listed in the Variables table.

Control surface names are case-sensitive and must match the CONTROL card name in the .avl file.

None  set_parameter(name: str, value: float) → None

Set a dimensional run-case parameter such as Mach number, velocity, or air density. See the Parameters table for accepted names.

None  set_constraint(variable: str, constraint: str, value: float) → None

Configure the Newton solver to drive variable so that constraint equals value. For example, to find the alpha that trims the pitching moment to zero: 

solver.set_constraint("alpha", "cm", 0.0)

See the Constraints table for all valid constraint names.

None  execute_run(max_iter: int = 20) → None

Run the Newton iteration loop. The solver iterates until all constraint residuals are below the convergence tolerance (2e-5) or max_iter iterations are reached. Results are written into solver.state and accessible via get_results().

Warning If results['converged'] is False, the solution did not reach tolerance. Increase max_iter or check that the flight conditions are physically reasonable.
dict  get_results() → dict[str, Any]

Return a dictionary of force, moment, and geometry results from the most recent solve. See the Results reference for all keys.

StabilityDerivatives  get_stability_derivatives() → StabilityDerivatives

Return classical stability-axis derivatives (dCL/dα, dCm/dα, stability-axis rate derivatives, and per-control dictionaries) from the most recent execute_run(). Derivatives with respect to alpha, beta, and control deflections are per radian; rate derivatives are non-dimensional. Default axis orientation is NASA Standard (X fwd, Z down). See Derivative coordinate systems.

ControlDerivatives  get_control_derivatives(axis: str = "stability") → ControlDerivatives

Return a control-only derivative matrix (one row per surface) from the most recent execute_run(). Pass axis="stability" (default) for columns CL, CD, CY, Cl, Cm, Cn, or axis="body" for CX, CY, CZ, Cl, Cm, Cn. Values are per radian of deflection. See Derivative coordinate systems.

BodyAxisDerivatives  get_body_axis_derivatives() → BodyAxisDerivatives

Return the AVL DERMATB-style body-axis derivative matrix (u–r and control rows; columns CX, CY, CZ, Cl, Cm, Cn). Control rows are per radian. Prefer get_control_derivatives(axis="body") when only control-surface rows are needed by name. See Derivative coordinate systems.

None  setup_trim(mode: int = 1) → None

Apply AVL trim presets. Mode 1: level flight or banked turn (airspeed and/or CL derived from mass and density). Mode 2: pull-up with turn radius. Does not run the aerodynamic solve — call execute_run() afterward.

EigenAnalysisResult  eigenvalues(use_approx: bool = False) → EigenAnalysisResult

Linearized rigid-body eigenanalysis at the current trimmed state. Returns mode names, frequencies, damping ratios, and time constants. Requires positive mass, inertia, and airspeed (typically from a .mass file). See Stability & Dynamics.

tuple[Figure, Axes]  plot_aircraft(**kwargs)

Plot the aircraft geometry in 3D. Delegates to openavl.plotting.plot_aircraft_3d. Does not require a prior solve. plot_geom() is an alias for plot_aircraft().

tuple[Figure, Axes]  plot_lift_distribution(**kwargs)

Plot spanwise strip lift (cl_lstrp) or normal-force loading (cnc) after execute_run(). Keyword arguments such as component, quantity, and show are forwarded to openavl.plotting.plot_lift_distribution.

tuple[Figure, Axes]  plot_cp(**kwargs)

Plot the solved pressure-coefficient distribution on lifting surfaces in 3D. Requires a completed execute_run() so that state.dcp is populated. Keyword arguments are forwarded to openavl.plotting.plot_cp.

Multi-constraint trim example

# Trimmed flight: zero pitching moment, zero rolling moment
# Solved by adjusting alpha and aileron deflection simultaneously
solver.set_constraint("alpha",    "cm",  0.0)
solver.set_constraint("aileron", "cmx", 0.0)
solver.execute_run(max_iter=40)

Stability & Dynamics

Beyond force coefficients, OpenAVL exposes AVL's stability-derivative extraction and linearized flight-dynamics eigenanalysis through AVLSolver.

Stability derivatives

solver.execute_run()
derivs = solver.get_stability_derivatives()

print(derivs.CL_a)   # dCL/d(alpha) per radian
print(derivs.Cm_a)   # dCm/d(alpha)
print(derivs.CL_q)   # dCL/d(pitch rate)
print(derivs.Cn_r)   # dCn/d(yaw rate)

# Control effectiveness (per radian of deflection)
print(derivs.Cm_d["elevator"])

# Control-only matrix in stability or body axes
ctrl = solver.get_control_derivatives(axis="stability")
print(ctrl.rows, ctrl.cols, ctrl.values)

StabilityDerivatives provides force/moment derivatives with respect to alpha, beta, stability-axis rates (p, q, r, non-dimensional), and each control surface. Derivatives with respect to alpha, beta, and control deflections are per radian. Control derivatives are stored in dictionaries keyed by control name (CL_d, Cm_d, Cl_d, Cn_d, etc.). For a control-only matrix in either axis system, use get_control_derivatives(axis=...).

Derivative coordinate systems

OpenAVL follows AVL’s axis conventions for forces, moments, and their partial derivatives. By default both stability- and body-axis derivatives use the NASA Standard orientation: X forward, Z down (AVL’s “Standard axis orientation”). This is controlled by solver.state.lnasa_sa, which defaults to True. Set it to False before execute_run() to use AVL’s Geometric convention (X aft, Z up).

AVL defines three axis systems for near-field forces. OpenAVL exposes derivative matrices for the first two; wind-axis derivatives are not provided (Trefftz-plane drag is available separately via CDFF in results).

SystemHow to obtainForcesMomentsPerturbations
Stability axes solver.get_stability_derivatives() CL, CD, CY Cl, Cm, Cn (roll/yaw in stability axes) α, β, p′, q′, r′, controls
Body axes solver.get_body_axis_derivatives() CX, CY, CZ Cl, Cm, Cn (body axes) u, v, w, p, q, r, controls
Control surfaces solver.get_control_derivatives(axis=...) axis="stability": CL, CD, CY, Cl, Cm, Cn; axis="body": CX, CY, CZ, Cl, Cm, Cn control deflections only (1/rad)
Wind axes Not exposed as derivatives Forces differ from stability axes only when β ≠ 0; use Trefftz CDFF for induced drag in performance work.

Stability axes are pitched by angle of attack from the body axes (y is common to both). Pitch moment Cm is identical in body and stability axes. Stability-axis roll and yaw moments (Cl, Cn in StabilityDerivatives) are transformed from body-axis moments; they are not the same as the body-axis Cl and Cn in the body derivative matrix.

Rate derivatives use non-dimensional rates (pb/2V, qc/2V, rb/2V). By default solver.state.lsa_rates is False, so trim constraints on body rates use body-axis p, q, r internally, while get_stability_derivatives() still reports stability-axis rate derivatives (p′, q′, r′). Set lsa_rates = True to interpret rate constraints in stability axes (AVL Options menu equivalent). 

body = solver.get_body_axis_derivatives()
# body.rows: ["u", "v", "w", "p", "q", "r", "d1", ...]
# body.cols: ["CX", "CY", "CZ", "Cl", "Cm", "Cn"]
# body.values[i][j] = d(col_j) / d(row_i)

ctrl = solver.get_control_derivatives(axis="body")
# ctrl.rows: control names; ctrl.cols: ["CX", "CY", "CZ", "Cl", "Cm", "Cn"]

get_results() mixes axis systems: CL, CD, and CY are stability-axis forces; CF is the raw body-axis force vector [Cx, Cy, Cz]. The body-axis derivative matrix follows AVL’s DERMATB sign layout and is not identical to differencing CF directly. Control-surface rows are returned per radian of deflection; velocity rows u, v, w are normalized perturbations (for small angles, v and w align with sideslip and angle of attack in radians).

Eigenvalue analysis

modes = solver.eigenvalues()

for mode in modes.modes:
    if mode.frequency_hz > 0.0:
        print(f"{mode.name}: freq={mode.frequency_hz:.3f} Hz, "
              f"damping={mode.damping_ratio:.3f}")
    elif mode.time_constant:
        print(f"{mode.name}: tau={mode.time_constant:.1f} s")

Each FlightMode record includes name, eigenvalue, frequency_hz, damping_ratio, and time_constant when applicable. Pass use_approx=True for AVL's faster approximate state matrix.

Prerequisite Eigenanalysis requires a converged trim with positive mass, inertia, and airspeed. Load a .mass file and set velocity (or let setup_trim compute it from target CL).

Visualization

AVLSolver exposes matplotlib-based visualization through plot_aircraft(), plot_lift_distribution(), and plot_cp(). The underlying implementations live in openavl.plotting and can also be called directly when plotting from an .avl path, Aircraft, or AVLModel without a solver instance. Matplotlib is installed by default with OpenAVL.

3D geometry preview

from openavl import AVLSolver

solver = AVLSolver("tests/data/avl/geometries/supra.avl")
solver.plot_aircraft()   # opens an interactive 3D window

# plot_geom() is an alias for plot_aircraft()
fig, ax = solver.plot_aircraft(show=False)
fig.savefig("supra.png")

# Or pass geometry directly via openavl.plotting
from openavl.plotting import plot_aircraft_3d
fig, ax = plot_aircraft_3d("plane.avl", show=False)

Surfaces are color-coded by name or component index; control-surface hinge locations are highlighted. This is a geometry wireframe preview — not a pressure (Cp) plot.

Spanwise lift distribution

After a converged solve, plot local strip lift (cl_lstrp) or normal-force loading (cnc) versus spanwise position: 

solver.execute_run()
solver.plot_lift_distribution(
    component=1,       # optional: restrict to one AVL component
    quantity="cl",     # "cl" or "cnc"
    title="Wing lift distribution",
)

# Or collect raw series for custom plots
from openavl.plotting import collect_lift_distribution

series = collect_lift_distribution(solver.state, solver.model, component=1)
for item in series:
    print(item.label, item.y, item.cl)

When a wing has clmax set via the Geometry API, the plot draws a dashed horizontal line at the limit.

Pressure coefficient (Cp)

After a converged solve, plot the lattice pressure distribution on lifting surfaces: 

solver.execute_run()
solver.plot_cp()                    # interactive 3D Cp plot
fig, ax = solver.plot_cp(show=False)  # save or customize
Note Use openavl.plotting.collect_cp_surfaces when you need the raw mesh and Cp arrays for custom visualization.

Geometry API

Build aircraft without .avl files using builder classes in openavl.geometry. Pass the resulting Aircraft directly to AVLSolver. 

from openavl import AVLSolver, Aircraft

ac = Aircraft(name="Demo", sref=10.0, cref=1.0, bref=8.0)
wing = ac.add_wing("Wing", n_chord=8, n_span=16, symmetric=True)
root = wing.add_section(xyzle=[0, 0, 0], chord=1.0)
root.set_airfoil_naca("2412")
tip = wing.add_section(xyzle=[0.2, 4, 0], chord=0.6)
tip.set_airfoil_naca("2412")
tip.add_control("aileron", xhinge=0.75, sgn_dup=-1.0)

solver = AVLSolver(ac, base_dir="/path/to/airfoils")

On any Section:

  • set_airfoil_naca("2412") — NACA 4-digit
  • set_airfoil_file("ag40d.dat") — external coordinate file (resolved via base_dir)
  • set_airfoil_coords([[x, y], ...]) — inline coordinates
Tip See examples/supra_geomapi_example.py for a full programmatic reproduction of the Supra sailplane, including multi-surface wings, bodies, and control surfaces.

CD(CL) Polars

Viscous strip drag uses AVL's piecewise-quadratic CD(CL) polar defined by three anchor points. In .avl files this is the CDCL keyword; in the Geometry API use CdclPolar. 

from openavl.geometry import CdclPolar

polar = CdclPolar.from_list([
    -1.0, 0.10,   # cl_neg, cd_neg
     0.0, 0.015,  # cl_min, cd_min  (cd_min > 0 enables viscous drag)
     1.5, 0.12,   # cl_pos, cd_pos
])
wing.set_cdcl_polar(polar)
# or per-section: section.set_cdcl_polar(polar)

CdclPolar.from_points() accepts three (CL, CD) tuples in any order (sorted internally). The polar is active when cd_min != 0 (polar.is_active).

Sectional CLmax (Stall Capping)

OpenAVL can limit sectional lift on each wing before forces are integrated. Set clmax on a Wing to the maximum local lift coefficient you expect that surface to sustain in attached flow. When any spanwise strip exceeds this value during a solve, its sectional forces and moments are scaled down so that the local lift coefficient equals clmax, while preserving force direction.

This is an OpenAVL extension: it is available only through the Geometry API. Legacy .avl files do not define clmax, so surfaces loaded from files behave as if clmax = 0 (capping disabled). 

wing = ac.add_wing("Wing", n_chord=8, n_span=16)
wing.clmax = 1.0   # 0.0 = disabled (default)

solver = AVLSolver(ac, alpha=12.0)
solver.execute_run()

# Inspect capped strip lift (local lift coefficient per strip)
j0 = int(solver.state.jfrst[0])
nj = int(solver.state.nj[0])
cl_local = solver.state.cl_lstrp[j0 : j0 + nj]
print(cl_local.max())   # <= clmax when capping is active

Behavior

  • Capping uses cl_lstrp, the local lift coefficient at each strip computed in the strip's local lift direction (including dihedral via the strip normal).
  • When cl_lstrp[j] > clmax, all strip force and moment components (cfstrp, clstrp, cdstrp, cystrp, cmstrp, and related local coefficients) are multiplied by clmax / cl_lstrp[j].
  • Integrated total CL, CD, and moments reflect the reduced sectional loads. Trefftz-plane far-field analysis is unchanged (it uses circulation directly, not the capped strip forces).
  • Stability-derivative sensitivities for stalled strips are not scaled; post-stall linear derivatives are not meaningful in a VLM context.

Solver output

When capping is applied during execute_run(), OpenAVL prints one message per affected surface per solve, for example: 

Local Cl > Clmax (1.000) on Wing: Clipping sectional lift force
Limitation Sectional CLmax capping is a simple magnitude limit, not a full stall model. It does not modify circulation or predict separated flow; use it as a practical estimate of how limiting peak sectional lift affects integrated aerodynamics.

OpenAVL reads the same .avl text format as the original Fortran code. This section summarizes the key keywords. For full details refer to the original AVL documentation by Drela & Youngren.

Mass File Format

An optional .mass file can be supplied to the AVLSolver constructor to specify mass and inertia properties. The format mirrors the original AVL mass file convention. 

#  mass file for My Aircraft
#  units: kg, m

Lunit = 1.0 m
Munit = 1.0 kg
Tunit = 1.0 s

#          x       y       z      mass
#  name
body        0.30  0.00  0.00   0.450

# Total mass and inertia tensor:
mass  = 0.450
Ixx   = 0.010
Iyy   = 0.008
Izz   = 0.015
Ixy   = 0.000
Ixz   = 0.001
Iyz   = 0.000
Note When a .mass file is supplied to AVLSolver, mass, CG, inertia tensor, and unit conversions are loaded into the solver state and used by setup_trim() and eigenvalues(). Constructor keyword arguments for flow parameters (rho, vel, xcg, etc.) are reapplied after mass-file defaults.

Parser API

The parser module is used internally by AVLSolver, but you can call it directly if you need access to the parsed geometry data structure. 

from openavl.fileio.parser import parse_avl_file, parse_avl, prepare_model
# or: from openavl import parse_avl_file, parse_avl

# Parse from file
model = parse_avl_file("path/to/plane.avl")

# Parse from a string
model = parse_avl(avl_text_string)

# Resolve AFILE paths relative to a base directory
model = prepare_model(model, base_dir="path/to/")

AVLState

AVLState (in openavl.core.state) is a dataclass that holds every NumPy array the solver needs. It replaces the COMMON blocks from the Fortran source.

You generally interact with AVLState through the AVLSolver wrapper. Direct access is available via solver.state when you need raw array data, for example to extract per-strip forces or stability derivative matrices after a solve.

Commonly accessed state fields

FieldShapeDescription
alfascalarAngle of attack in radians.

machscalarFreestream Mach number.
cltotscalarTotal CL after solve.
cdtotscalarTotal CD after solve.
cytotscalarTotal CY after solve.
cmtot(3,)Rolling, pitching, yawing moment coefficients.
clffscalarFar-field CL (Trefftz plane).
